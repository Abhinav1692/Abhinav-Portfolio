Most Used "HTTP Response Codes" which you should know.

Hello, I have listed all the most used HTTP response codes and their meanings which we often see. HTTP response status codes are basically used by server in response to a client's request. The first digit of the code signifes one of the standard classes of response. HTTP status codes are categorised into 5 classes which are as follows.

1xx "Informational Response"

100 "Continue"

This response shows that the server received the request headers and the client should start to send request body or, ignore the response if the request is already finished.

101 "Early Hints"

it is used to return some response headers before the final HTTP message.

2xx "Successful Responses"

200 "Ok"

Standard response for successful HTTP requests.

201 "Created"

This response shows that the request has been fulfilled and it results in the creation of new resource.

202 "Accepted"

The request has been accepted but not yet acted upon.

3xx "Redirection Messages"

300 "Multiple Choice"

This response shows that the request has more than one response from which the client may choose.

301 "Moved Permanently"

The URL of the requested resource has been changed permanently. Therefore, New URL is given in the response.

4xx "Client Error Responses"

400 "Bad Request"

The server is not able to understand the request due to invalid syntax.

403 "Forbidden"

The client doesn't have the right to access the content. Therefore, Server refused to provide the requested resource. However, the Client's identity is known to the server.

404 "Not Found"

The server fails to find the requested resource. It means in the browser, The URL is not recognized.

5xx "Server Error Messages"

500 "Internal Server Error"

An unexpected situation has occurred which server doesn't know how to handle.

503 "Service Unavailable"

This response shows that the Server is not ready to handle the client's request. It may be because of the "server down for maintenance" or it got overloaded.
